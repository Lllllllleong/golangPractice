## üîç Finding Solutions

### Search Tips
1. **By Problem Name**: Use Ctrl+F (or Cmd+F) and search for keywords from the problem title
2. **By Data Structure**: Search for class names like `TreeNode`, `ListNode`, `Graph`
3. **By Pattern**: Search for algorithm keywords like `DFS`, `BFS`, `DP`, `binary search`

### File Organization
- Solutions are added chronologically to Practice files
- New Practice file created when current file reaches ~4,000 lines (IDE performance threshold)
- No specific difficulty ordering - problems mixed within files

## üí° Usage Tips

### Running Solutions
```bash

# Run if there's a main method (rare - mostly just solution functions)

```

### Testing
- Most functions are standalone solutions (no main method)
- Copy function to LeetCode editor to test
- Or create your own test harness in a separate file

### Adding New Solutions
1. Add to `Practice6.java` (most recent file)
2. Include comment block with problem name and complexity
3. When file approaches 4,000 lines, create `Practice7.java`
